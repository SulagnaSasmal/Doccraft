# DocCraft Flow Architecture

## State 1: The Inbox (Dashboard)
- Primary Action: Upload/Drop.
- Goal: Get files into the system.

## State 2: The Review (Workspace)
- Primary Action: View/Read.
- Sidebar: File Outline + Metadata.
- Goal: Understand the document content.

## State 3: The Process (Tool Overlay)
- Primary Action: Split/Merge/OCR.
- UI: Appears as a 'Sheet' or 'Modal' over the Workspace.
- Goal: Transform the document without leaving the context.

## State 4: The Archive (History)
- Goal: Access previous versions and download logs.
